Definitions of One-to-One and Onto

First review the following definitions from class.







Example

Define linear transformation T(x⃗)=Ax⃗T(\vec{x}) = A\vec{x}T(x)=Ax where A=[100011]. A = \left[ \begin{array}{ccc} 1 & 0 & 0 \\ 0 & 1 & 1 \end{array} \right]. A=[10​01​01​]. Is TTT onto? One-to-one?

TTT is onto since for any b⃗∈R2\vec{b} \in \mathbb{R}^2b∈R2, T(x⃗)=Ax⃗=b⃗T(\vec{x})=A\vec{x}=\vec{b}T(x)=Ax=b yields the augmented matrix (already in row echelon form) [100b1011b2]. \left[ \begin{array}{ccc|c} 1 & 0 & 0 & b_1 \\ 0 & 1 & 1 & b_2 \end{array} \right]. [10​01​01​b1​b2​​]. which has a full set of pivots and hence always has a solution.

TTT is not one-to-one since the augmented matrix has a free variable implying that multiple x⃗\vec{x}x can be mapped to the same b⃗\vec{b}b. For example, we have that both T(0⃗)=0⃗T(\vec{0})=\vec{0}T(0)=0 and T([0,1,−1]T)=0⃗T([0,1,-1]^T)=\vec{0}T([0,1,−1]T)=0.

Part 2: Difference equations, migration application, Exercises 2 and 3

Here we the idea of a difference equation:

  • Formulated as a linear transformation

  • Applied to solve population migration

Difference equation definition

A linear difference equation generates a sequence of vectors x0⃗,x1⃗,x2⃗,…\vec{x_0}, \vec{x_1}, \vec{x_2}, \dotsx0​​,x1​​,x2​​,… by iterating the matrix transformation T(x⃗)=Ax⃗T(\vec{x})=A\vec{x}T(x)=Ax as x⃗k+1=Axk⃗. \vec{x}_{k+1} = A \vec{x_k}. xk+1​=Axk​​. That is, for x0⃗\vec{x_0}x0​​ given, x1⃗=Ax0⃗,x2⃗=Ax1⃗,,x3⃗=Ax2⃗,… \vec{x_1} = A\vec{x_0}, \quad\quad \vec{x_2}=A\vec{x_1},, \quad\quad \vec{x_3}=A\vec{x_2}, \quad\quad\dots x1​​=Ax0​​,x2​​=Ax1​​,,x3​​=Ax2​​,…

popSimulator <- function(x0,n){
    A <- matrix(c(0.93, 0.02,
                 0.07, 0.98), nrow=2, ncol=2, byrow=TRUE)
    x <- x0
    for (i in c(1:n)){
        x <- A %*% x
        print(paste("Year: ", i))
        print(x)
    }
    
    for (i in c((n+1):100)){
        x <- A %*% x
    }
    
    print("Long term projection")
    return(x)
}
